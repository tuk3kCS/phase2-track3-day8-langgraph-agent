02 / TIMELINE

Workflow Execution Lifecycle

Chronological lifecycle of a support ticket query running through the LangGraph agent.

Step 1

User Request

Customer submits support ticket query (e.g. "Refund order #12345") via client UI/CLI.

Step 2

State Initialization

`initial_state` triggers, allocating thread config, normalizing queries, and creating list reducers.

Step 3

Routing Decision

`classify_node` executes LLM structured output classification to map queries to distinct routes (simple, tool, risky, etc.).

Step 4

Tool Execution

Appropriate mock tools are triggered based on routes (e.g., searching DB for order lookup details).

Step 5

LLM Reasoning

LLM-as-judge assesses tool results. For risky routes, OpenAI models evaluate context and actions.

Step 6

Human Approval

If the query is flagged as high-risk, the node calls `interrupt()` to save state and pause execution for human verification.

Step 7

State Update

Resumed inputs or tool logs append to the state lists, updating execution metrics and state history checkpoints.

Step 8

Final Response

Response generated by the grounded LLM is returned to the user, and the thread state is finalized and closed.
